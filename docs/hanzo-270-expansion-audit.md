## Acceptance rule and method

A row is accepted here only when all of the following exist together:

1. an exact timestamped YouTube watch URL (or an exact Reddit comment permalink; none qualified here);
2. an exact preserved quote;
3. directly reviewed visual evidence of a concrete, frame-visible point/sightline, or a continuously visible bounded route;
4. directly established map identity, without selection from chapter order, architecture alone, a generic claim, or an unseen interval.

The audit followed **retrieve → verify → parse** for saved sources: the previously retrieved metadata/caption/storyboard artifacts and their recorded hashes were checked in the source audits before their parsed claims were used here. Primary inputs were `docs/production-prompt-audit-2026-07-22.md`, `docs/hanzo-visual-audit-2026-07-21.md`, `docs/hanzo-transcript-shortlist-2026-07-21.md`, `docs/hanzo-source-cover-2026-07-21.md`, and the raw captions, metadata, MHTML, checksums, and 104 saved `sb1` sheets under `/opt/data/annotated-maps/hanzo-source-cover/`. Existing direct-search JSON was treated only as discovery output, never as placement evidence. No unfamiliar repository code was run.

“Accepted candidate” below means evidence-defensible for a later production decision; it does **not** claim that imagery, coordinate placement, objective phase, or application integration has been completed. Prompts are narrowed to what the reviewed frame actually shows.
